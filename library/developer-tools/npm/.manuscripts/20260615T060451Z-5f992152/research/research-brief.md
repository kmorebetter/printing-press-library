# npm package intelligence print research

Source: public npm registry and downloads APIs.

The promoted print was shaped around three agent-facing workflows:

- `package`: summarize one package with latest version, maintainers, dependency counts, release freshness, and last-month downloads.
- `compare`: rank packages by adoption and maintenance signals.
- `risk`: turn registry metadata into explicit package risk signals.

No credentials are required for the public read-only workflows.

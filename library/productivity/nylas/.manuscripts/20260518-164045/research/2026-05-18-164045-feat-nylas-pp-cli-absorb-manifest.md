## Transcendence

Hand-coded features only possible with our local SQLite mirror + cross-grant aggregation. See `novel-features-subagent-output.md` for full candidate list, kill rationale (5/18 killed), and customer model.

### Local state that compounds
| # | Feature | Command | Buildability | Why Only We Can Do This |
|---|---------|---------|--------------|------------------------|
| T1 | sync | `nylas-pp-cli sync --grant all --resource messages,events --since 24h` | hand-code | Local SQLite + per-(grant, resource_type) cursors; incumbent is stateless |
| T2 | since | `nylas-pp-cli since 2h --resource messages --json` | hand-code | Answers from the mirror across all grants, no API hit |
| T3 | search | `nylas-pp-cli search "invoice overdue" --grant all --since 30d` | hand-code | FTS5 over messages/threads/events/contacts spanning grants |
| T4 | export | `nylas-pp-cli export --resource messages --since 90d --format ndjson` | hand-code | Streams the store to ndjson/parquet for downstream analytics |

| # | Feature | Command | Buildability | Why Only We Can Do This |

**Hand-code total:** 13 transcendence features. Largest (`sync`, `mcp local-tools annotations`) ~150 LoC each; smallest (`since`, `export`) ~50 LoC.

**Scope cuts (killed by adversarial pass, see subagent output):**
- inbox-diff — folds into `sql` + `export` once snapshots can be exported
- rule-audit — punts to a future release; risks >150 LoC of baseline schema
- batch — generic "run a file of commands" is shell territory
- meeting-load — partially covered by `sql` over events
- thread-thread — narrow; `search` + `sql` reach it
